ObsPy is a community-driven, open-source project dedicated to provide a Python framework for processing seismological data. It provides parsers for common file formats, clients to access data centers and seismological signal processing routines which allow the manipulation of seismological time series (see Beyreuther et al. 2010, Megies et al. 2011).

The goal of the ObsPy project is to facilitate rapid application and workflow development for seismology.

Features

Read and Write Various Waveform Formats

ObsPy enables you to painlessly read and write all commonly used waveform data formats with a unified interface. It automatically detects a file's format meaning that you never need to worry about the format of your data again. Simply combine data from heterogeneous sources and get straight to work.

We currently support importing waveform data in MiniSEED, SAC, SEGY, GSE2, GSE1, DATAMARK, CSS, SEISAN, SEG2, SEED, WAV, and the SeismicHandler formats. Most formats can also be exported again. The employed plug-in architecture even enables you to add support for your own custom formats.

Webservice and Database Access Clients

ObsPy offers clients for accessing data from the biggest and most often used datacenters. No matter the origin of the data, the processing within ObsPy does not differ, enabling the easy creation of workflows combining heterogeneous data sets.

It currently supports data retrieval from IRIS, Orfeus, NERIES, and NEIC datacenters and the Earthworm and SeisHub databases. All clients support waveform and oftentimes also station and event metadata downloading, depending on the backend.

Comprehensive Signal Processing Library

ObsPy contains a fairly comprehensive seismological signal processing library. It aims to offer all commonly used and agreed upon processing functionality.

It contains various filtering, triggering, rotation, array analysis, instrument correction and simulation, time frequency and many more routines with and intuitive interface.
